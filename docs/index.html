// HOW IT WORKS

The RL Training Loop

At each simulation step, the agent observes the intersection state, picks a traffic phase, receives a reward, and learns from experience.

01
Observe State
80-dimensional binary vector encoding vehicle positions across 8 lanes × 10 cells each
02
Choose Action
ε-greedy policy selects one of 4 signal phases — exploring early, exploiting later
03
Get Reward
Reward = −Δ cumulative waiting time. Less waiting = positive signal to the agent
04
Learn
Bellman equation updates Q-values using random batches from the experience replay buffer

// ARCHITECTURE

Deep Q-Network Design

The agent uses a 3-layer fully connected network mapping the 80-dim state to Q-values for each of the 4 possible traffic phases.

STATE REPRESENTATION

80-Dimensional Binary State

8 lanes × 10 position cells. Each cell is 1 if a stationary vehicle occupies it, 0 otherwise. Cells closer to the intersection are smaller (7m) while cells farther away are larger (400m).

REWARD FUNCTION

Negative Waiting Time Delta

reward = old_cumulative_wait − new_cumulative_wait. Negative when congestion grows, positive when it shrinks. This directly optimises the metric we care about.

EXPERIENCE REPLAY

50,000-Sample Memory Buffer

Stores (s, a, r, s') tuples. Random batch sampling of 100 transitions per update breaks temporal correlations and stabilises training significantly.

EXPLORATION

ε-Greedy Decay

Starts at ε=1.0 (fully random) and decays to ε=0.01 over 400 episodes. Early random exploration builds diverse memory; later exploitation uses learned Q-values.
